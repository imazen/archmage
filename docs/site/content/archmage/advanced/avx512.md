AVX-512 provides 512-bit vectors and advanced features. Here's how to use it effectively with archmage.

## Enabling AVX-512

Add the feature to your `Cargo.toml`:

```toml
[dependencies]
archmage = { version = "0.8", features = ["avx512"] }
```

## AVX-512 Tokens

| Token | Features | CPUs |
|-------|----------|------|
| `X64V4Token` | F, BW, CD, DQ, VL | Skylake-X, Zen 4 |
| `X64V4xToken` | + VNNI, VBMI, IFMA, etc. | Ice Lake+, Zen 4+ |
| `Avx512Fp16Token` | + FP16 | Sapphire Rapids |

Aliases:
- `Server64` = `X64V4Token`
- `Avx512Token` = `X64V4Token`

## Basic Usage

```rust
use archmage::{X64V4Token, SimdToken, arcane};

#[arcane(import_intrinsics)]
fn sum_of_squares(token: X64V4Token, data: &[f32; 16]) -> f32 {
    let v = _mm512_loadu_ps(data.as_ptr());
    let sq = _mm512_mul_ps(v, v);
    _mm512_reduce_add_ps(sq)
}

fn main() {
    if let Some(token) = X64V4Token::summon() {
        let data = [1.0f32; 16];
        let result = sum_of_squares(token, &data);
        println!("Result: {}", result);
    } else {
        println!("AVX-512 not available");
    }
}
```

## 512-bit Intrinsic Types

| Type | Elements | Description |
|------|----------|-------------|
| `__m512` | 16 x f32 | 512-bit float |
| `__m512d` | 8 x f64 | 512-bit double |
| `__m512i` | varies | 512-bit integer |

## Masking

AVX-512's killer feature is per-lane masking:

```rust
#[arcane(import_intrinsics)]
fn masked_add(token: X64V4Token, a: __m512, b: __m512, mask: __mmask16) -> __m512 {
    // Only add lanes where mask bit is 1
    // Other lanes keep value from `a`
    _mm512_mask_add_ps(a, mask, a, b)
}
```

## Tiered Fallback with AVX-512

```rust
use archmage::{incant, arcane};

pub fn process(data: &mut [f32]) {
    incant!(process(data), [v4, v3, neon, wasm128])
}

#[arcane(import_intrinsics)]
fn process_v4(token: X64V4Token, data: &mut [f32]) {
    for chunk in data.chunks_exact_mut(16) {
        let v = _mm512_loadu_ps(chunk.as_ptr());
        let result = _mm512_mul_ps(v, v);
        _mm512_storeu_ps(chunk.as_mut_ptr(), result);
    }
    // Handle remainder with AVX2 (V4 can downcast to V3)
    let remainder = data.chunks_exact_mut(16).into_remainder();
    if !remainder.is_empty() {
        process_v3(token, remainder);  // Downcast works!
    }
}

#[arcane(import_intrinsics)]
fn process_v3(token: X64V3Token, data: &mut [f32]) {
    for chunk in data.chunks_exact_mut(8) {
        let arr: &mut [f32; 8] = chunk.try_into().unwrap();
        let v = _mm256_loadu_ps(arr as &[f32; 8]);
        let result = _mm256_mul_ps(v, v);
        _mm256_storeu_ps(arr, result);
    }
    for x in data.chunks_exact_mut(8).into_remainder() {
        *x = *x * *x;
    }
}

fn process_scalar(data: &mut [f32]) {
    for x in data.iter_mut() {
        *x = *x * *x;
    }
}
```

## AVX-512 Performance Considerations

### When AVX-512 Wins

- **Large data**: Processing 16 floats vs 8 is 2x work per instruction
- **Masked operations**: No equivalent in AVX2
- **Gather/scatter**: Much faster than AVX2's versions
- **Specific instructions**: VPTERNLOG, conflict detection, etc.

### When AVX2 Might Win

- **Memory-bound code**: Wider vectors don't help if you're waiting for RAM

## Checking for AVX-512

```rust
use archmage::{X64V4Token, SimdToken};

fn check_avx512() {
    match X64V4Token::compiled_with() {
        Some(true) => println!("Compile-time AVX-512"),
        Some(false) => println!("Not x86-64"),
        None => {
            if X64V4Token::summon().is_some() {
                println!("Runtime AVX-512 available");
            } else {
                println!("No AVX-512");
            }
        }
    }
}
```
